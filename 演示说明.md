# QV 监控 PWA - 演示模式使用说明

## 快速开始

1. **打开演示模式**：在首页顶栏右侧，点击 **"演示模式"** 开关
2. **系统会自动添加一个演示设备**：名为"演示设备（客厅）"
3. **点击"查看演示设备"**：进入设备详情页

## 功能演示

### 设备列表页 (`/`)
- 顶部切换演示模式开关
- 演示设备卡片显示在线状态
- 可点击设备进入详情

### 设备详情页 (`/device/:id`)
- **状态总览**：设备信息、系统信息、存储状态
- **网络信息**：IP地址、连接方式、信号强度
- **RTSP 地址**：点击复制按钮，附 VLC/ffplay 播放说明
- **操作菜单**：设置、重启、删除设备
- **刷新按钮**：模拟重新获取设备状态

### 设备配置页 (`/device/:id/config`)
- **时间设置**：时区和时间显示
- **图像翻转**：开关控制
- **音量调节**：推子控制
- **网络配置**：静态/DHCP、IP、子网掩码、网关、DNS
- **帧率模式**：低/正常/高/超级
- **报警设置**：移动检测、声音检测、灵敏度
- **录像计划**：按天设置时间段

## 模拟数据特点

- **延迟模拟**：API 调用有 300-1500ms 延迟，模拟真实网络
- **控制台日志**：所有操作在浏览器控制台有日志输出
- **状态持久化**：设备列表保存在 localStorage（Base64 编码）
- **实时响应**：配置更改会即时反映在 UI 上

## 测试建议

1. **添加设备流程**：关闭演示模式 → 点击"添加设备" → 填写任意表单 → 提交
2. **配置操作**：进入设置页 → 修改任意参数 → 保存 → 观察 UI 变化
3. **RTSP 测试**：复制 RTSP 地址 → 用 VLC 播放器测试（需要网络可达）

## 切换到真实模式

1. 获取 QV 平台的 **API Key ID** 和 **API Key Secret**
2. 获取设备 **Device ID**
3. 关闭演示模式开关
4. 使用真实密钥添加设备
5. 确保网络可以访问设备 IP 地址

## 技术架构

- **前端**：React + TypeScript + Vite
- **UI 框架**：Material-UI (MUI)
- **路由**：React Router
- **状态管理**：React Context + useReducer
- **PWA**：Vite PWA 插件，可安装到手机桌面
- **模拟 API**：MockDeviceApiClient + MockCloudApiClient

## 常见问题

**Q: 演示模式下的设备状态是固定的吗？**
A: 是的，但操作（重启、设置等）会在控制台输出日志。

**Q: 可以添加多个演示设备吗？**
A: 可以，点击添加按钮，随意填写表单即可。

**Q: RTSP 地址是真的吗？**
A: 只是示例格式，实际需要真实设备才能播放。
